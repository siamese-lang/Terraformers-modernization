# ADR-0019: Maintain an Explicit Status Document

## Status

Accepted

## Context

The repository will evolve through several migration phases. Without a status document, it can become unclear which parts are complete and which are planned.

## Decision

Maintain `docs/status.md` as the current state summary.

## Consequences

- Readers can distinguish completed baseline work from future imports.
- The project avoids overclaiming.
- Status must be updated after backend, infrastructure, workflow, or evidence changes.
